# 05. Handoff Policy

## Статус

Supporting-doc для `v2 handoff gate`.
Handoff в `v2` остаётся explicit workflow-owned действием, даже если инициирующий сигнал исходит от agent core.

## Принцип

Handoff — не аварийная капитуляция системы, а штатный путь передачи кейса человеку.

Agent может:

- рекомендовать handoff;
- объяснять, почему он нужен;
- подготовить draft summary.

Но финальный handoff выполняет внешний workflow gate.

## Trigger Classes

### Явные user triggers

- пользователь просит человека;
- пользователь просит менеджера или звонок;
- пользователь явно не хочет продолжать диалог с ботом.

### System / safety triggers

- низкая уверенность по критичному вопросу;
- weak/empty retrieval в важном knowledge-driven кейсе;
- превышен лимит meaningful уточнений;
- сбор контакта/лида застрял;
- требуется high-impact действие, не разрешённое агенту автономно;
- интеграционная ошибка ломает нормальное продолжение сценария.

## Что должен сделать handoff gate

1. Зафиксировать причину handoff.
2. Построить `handoff-summary-v2`.
3. Записать internal note / operator signal.
4. Отправить пользователю явное сообщение о передаче.
5. Остановить bot-owned continuation этого кейса.

## Требования к handoff summary

Summary должен содержать:

- причину handoff;
- краткое описание запроса пользователя;
- текущую стадию диалога;
- уже собранные поля / ограничения / ожидания;
- агентскую рекомендацию следующего шага, если она полезна;
- trace context, достаточный для входа оператора без потери картины.

## Запреты

- нельзя скрывать handoff под видом "я уточню и вернусь", если реального handoff нет;
- нельзя возвращать кейс боту после explicit handoff без отдельной политики;
- нельзя формировать summary без явной причины и контекста;
- нельзя считать agent recommendation самим handoff-событием.

## Acceptance-oriented признаки правильного handoff

- причина handoff понятна;
- пользователь видит честное сообщение о передаче;
- оператор получает пригодный summary;
- trace показывает, почему именно этот кейс ушёл человеку;
- бот не продолжает скрытое владение кейсом после передачи.
