# Phase 0 Bootstrap Runbook

## Цель

Подтвердить, что репозиторий готов к началу Фазы 1 без архитектурного дрейфа.

## До начала Фазы 1

1. Прочитать `tasks/codex-initiating-prompt.md` и использовать его как стартовый prompt.
2. Проверить наличие обязательных каталогов и bootstrap-артефактов.
3. Убедиться, что Phase 0 создала placeholders, а не скрытую реализацию workflow.
4. Убедиться, что `.codex/` не создавался без явной проектной необходимости.

## Проверки

1. Контракты в `contracts/` читаются и остаются каноническим runtime baseline.
2. Prompt-файлы в `prompts/` существуют и описывают только intent/ограничения будущих фаз.
3. Workflow JSON в `workflows/` являются placeholder-only и не содержат runtime-бизнес-логики.
4. `samples/` содержит опорные payloads для chatwoot, leads и rag.
5. `ops/env.example` не содержит секретов и задаёт только baseline-переменные среды.

## Что считается нарушением

- попытка реализовать Phase 1+ внутри bootstrap-артефактов;
- появление `Code`, `Function`, `Function Item` или их аналогов;
- создание repo-local `.codex/` без явного обоснования;
- расхождение файловых артефактов с master-spec без фиксации `SPEC_GAP`.

## Выход

Если все пункты выше выполнены, можно начинать Фазу 1 `Inbound Echo`.
